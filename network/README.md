## 1. HTTP와 HTTPS의 차이점은 무엇인가?

- HTTP와 HTTPS는 인터넷에서 데이터를 주고 받는데 사용하는 프로토콜입니다.
- 둘의 가장 큰 차이점은 보안성, 무결성 입니다.
- HTTP는 데이터를 암호화하지 않고 전송합니다. 따라서 중간에 데이터가 탈취 됐을 시, 악용될 여지가 있습니다. <br />
또한 데이터가 전송 중에 변조, 손상되지 않았음을 보장하지 않습니다.(무결성 보장 X)
- HTTPS는 SSL, TLS 프로토콜을 사용하여 데이터를 암호화합니다. 이를 통해 데이터의 보안성을 높이고 무결성도 보장할 수 있습니다. <br />
또한 SSL, TLS 인증서를 통해 웹 사이트의 신원을 인증할 수 있습니다. 이를 통해 사용자는 웹 사이트가 신뢰할만 한 곳인지 확인할 수 있습니다. <br />

<br />

### 1-1. 프로토콜이란 무엇인가?

- 프로토콜은 서로 다른 시스템 간에 데이터를 주고받는 데 사용되는 약속된 규칙의 집합입니다.
- 데이터가 각 단계에서 어떤 정보가 전송되어야 하는지, 어떤 형식으로, 어떠한 암호화 방식을 사용해야 하는 지 등을 정의합니다.

<br />

### 1-2. SSL과 TLS 프로토콜은 무엇인가?

- SSL과 TLS는 데이터를 보호하기 위한 암호화 프로토콜입니다.
- TLS는 SSL를 대체하기 위해 개발된 프로토콜 입니다.
- 서버는 인증서를 클라이언트에 제공하고, 클라이언트는 신뢰도 확인 및 인증서에 제공된 공개키를 통해 데이터를 암호화하여 전송합니다. <br />
서버는 클라이언트에서 전송된 데이터를 비공개키를 통해 데이터를 복호화합니다. 이러한 방식을 공개키 암호화 방식 혹은 비대칭키 암호화 방식이라 말합니다.

## 2. HTTP 요청 메소드는 무엇인가?

- HTTP 요청 메서드는 클라이언트가 서버로 요청을 보낼 때, 요청의 목적과 방식을 지정하는 데에 사용되는 메서드입니다.
- GET: 서버의 데이터를 조회할 때 사용하는 메서드 입니다.
- POST: 서버에 데이터를 추가하고자 할 때 사용 합니다.
- PUT : 서버의 데이터를 업데이트 하고자 할 때 사용합니다. (전체 데이터를 업데이트 합니다.)
- PATCH: 서버의 데이터 중 일부분만 업데이트 하고자 할 때 사용합니다.
- DELETE: 서버의 데이터를 삭제하고자 할 때 사용합니다.

<br />

### 2-1. 쿼리(query)와 파라미터(params)의 차이는 무엇인가?

- 쿼리와 파라미터는 공통적으로 서버에 특정 조건을 전달하는 매개 변수입니다.
- 쿼리 : 주로 GET 요청에서 사용됩니다. "key=value" 형식으로 URL의 끝 부분에 "?"와 함께 추가됩니다. <br />
데이터를 검색할 때 사용 됩니다. <br />
ex) https://www.example.com/search?q=test
- 파라미터 : "/"를 사용해 URL의 끝 부분에 추가 됩니다. <br />
서버에서 특정 조건을 요청할 때 사용 됩니다. 주로 RESTful API 에서 사용됩니다. <br />
예를 들어, 파라미터를 포함하면 특정 데이터를 삭제 한다는 의미이고, 파라미터가 없다면 특정 데이터를 생성 하는 방식으로 구성할 수 있습니다. <br />
ex) https://www.example.com/items/1234

<br />

### 2-2. URL은 무엇인가? 

- URL은 인터넷 상에서 특정한 리소스의 위치를 가리키는 문자열 입니다.
- URL은 다음과 같이 구성됩니다.
```
scheme://host:port/path/?=query=value#fragment
```
- 스키마(scheme): 프로토콜을 의미합니다.
- 호스트(host): 리소스가 위치한 서버의 도메인 이름 또는 IP 주소를 나타냅니다.
- 포트(port): 호스트와 통신할 때 사용되는 포트 번호를 의미합니다. HTTP의 기본 포트 번호는 80, HTTPS의 기본 번호는 443 입니다. 생략 가능합니다.
- 경로(path): 리소스의 경로를 나타냅니다.
- 쿼리 문자열: 위 설명과 같습니다.
- 프래그먼트: 문서 내의 특정 위치를 나타냅니다.

<br />

### 2-3. 도메인이란 무엇인가?

- 인터넷 상에서 컴퓨터들을 식별하기 위한 이름으로 구성된 계층 구조적인 이름체계 입니다. 
- 각 컴퓨터는 고유한 IP 주소를 가지고 있지만 이를 사람이 기억하기 어렵고, 변경 될 수 있기 때문에, 도메인 이름을 사용합니다.

<br />

### 2-4. IP란 무엇인가?

- 인터넷 프로토콜의 약자로, 인터넷 상에서 컴퓨터들이 서로 정보를 주고 받을 수 있게 하는 프로토콜입니다. 일종의 컴퓨터의 주소라고 볼 수 있습니다. <br />
- IPv4 (32bit 주소 체계), IPv6(128bit 주소 체계)가 있습니다.
