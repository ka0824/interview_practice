## 1. React란 무엇인가?

- 페이스북에서 개발한 UI 라이브러리 입니다.
- 데이터를(위에서 아래로) 관리합니다.
- JSX 문법을 사용하여 HTML과 자바스크립트 코드를 함께 작성 가능합니다.

<br />

### 1-1. 리액트는 왜 프레임워크가 아니라 라이브러리인가?

- 리액트는 화면의 뷰만 담당합니다. 따라서 전역 상태 관리, 라우팅 등은 정해진 방법이 아니라 다양한 라이브러리로 각자 다르게 구성 가능합니다. <br />

<br />

### 1-2. 리액트와 뷰(Vue)는 무슨 차이가 있는가?

- 뷰는 프레임 워크입니다. 뷰는 리액트에 비해 자체적으로 앱에 대한 더 많은 제어가 가능합니다. 리액트와는 달리 자체적인 제어 방법을 제공하므로 <br />
코드의 형식이 어느 정도 정형화 되어있다고 볼 수 있습니다. 리액트에 비해 자유도는 떨어진다고 볼 수 있지만, 학습 곡선이 낮고, 코드의 통일성을 확보 <br />
하기 쉽다는 이점이 있다고도 볼 수 있습니다.

<br />

### 1-3. JSX란 무엇인가?

- 리액트에서 UI를 구성하는 요소를 정의하는 문법입니다. 
- JSX로 컴포넌트를 사용하여 UI를 구성하면, 이를 가상 돔(Virtual Dom)으로 변환한 후 실제 돔으로 반영합니다.
- 이 때에, 가상 돔에서 변경이 일어난 부분을 감지한 뒤 변경된 부분만 다시 그려 성능을 향상시킵니다.

## 2. state와 props의 차이점은 무엇인가?

- state는 컴포넌트 내에서 관리되는 데이터 입니다. <br />
state가 변경되면 해당 컴포넌트와, 해당 컴포넌트의 하위 컴포넌트가 모두 리렌더링 됩니다. <br />
state에는 api 호출, 사용자 입력 등에 동적으로 값이 변경되는 데이터를 다룹니다. <br />
상위 컴포넌트에서 state가 변경 되나 하위 컴포넌트가 해당 state에 영향을 받지 않는다면 React.memo()를 사용하여 리렌더링을 막을 수 있습니다. <br />

- props는 부모 컴포넌트로 부터 넘겨 받은 읽기 전용 데이터 입니다. <br />
부모의 state를 props로 받았다면, 해당 state가 변경될 때마다 현재 컴포넌트도 리렌더링 됩니다. <br />

<br />

### 2-1. 상위 컴포넌트로 데이터를 넘겨주고 싶을 때는 어떻게 하는가?

- 상위 컴포넌트로 값을 넘겨주기 보다는 state를 위치를 공통적으로 사용하는 컴포넌트의 상위 컴포넌트로 옮기는 방법을 생각해 볼 수 있습니다. <br />
다만, 이러한 방법을 사용할 경우, props로 값을 내려 받을 수 있도록 코드를 수정해야 합니다. 이러한 경우 useContext 훅을 사용을 고려해 볼 필요가 있습니다. <br />
useContext 상태에 대한 접근을 좀더 용이하게 만들어 줍니다.

- 혹은 redux 등 전역 상태 관리 라이브러리를 사용해 볼 수 있습니다. 전역 상태 관리 라이브러리를 사용할 경우, 데이터의 단방향 흐름에서 벗어나기 때문에 <br />
데이터 설계에 대한 고민을 좀 더 줄일 수 있습니다.

<br />

## 3. lifecycle 메소드는 무엇이 있는가?

- 클래스형 컴포넌트에서는 컴포넌트가 처음 그려질 때, 사라질 때 업데이트 될 때 등에 해당하는 메서드가 따로 있었지만, <br />
함수형 컴포넌트에서는 useEffect(), useLayoutEffect()를 통해 라이프사이클 이벤트를 처리할 수 있습니다.

- useEffect() 메서드를 사용해 라이프 사이클을 제어하는 코드는 다음과 같습니다.
```
useEffect(() => {

  컴포넌트가 처음 그려질 때, 렌더링의 기준이 되는 값이 변할 때 실행할 코드들
  
  return () => {
    컴포넌트가 제거될 때 실행될 코드
  }

}, [렌더링의 기준이 되는 값]) // 렌더링의 기준이 되는 값이 변하면 useEffect내의 코드가 실행됩니다. 만약 빈배열일 경우 컴포넌트가 처음 그려질 때만 코드가 실행됩니다.
```

<br />

### 3-1. useEffect() 메서드와 useLayoutEffect() 메서드의 차이는 무엇인가?

- 두 메서드의 차이는 실행 시점, 실행방식에서 차이가 있습니다.
- useEffect()는 브라우저의 렌더링 작업이 모두 끝난 이후에 실행됩니다. 비동기적으로 실행됩니다.
- useLayoutEffect()는 리액트가 DOM을 업데이트하기 전에 실행됩니다. 동기적으로 실행됩니다.
- useLayoutEffect()는 렌더링 하기 직전에 함수를 동기적으로 실행시키므로, 렌더링이 늦춰질 우려가 있습니다. 따라서 DOM을 제어해야 하는 경우에만 useLayoutEffect() <br />
을 사용하는 것이 좋습니다.

## 4. 조건부 렌더링은 어떻게 구현하는가?

- 삼항 연산자 혹은 && 연산자를 이용해 구현할 수 있습니다.
- 삼항 연산자를 이용할 경우는 다음과 같습니다.
```
<div>
  {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
</div>
```

- && 연산자를 이용할 경우는 다음과 같습니다.
```
<div>
  {isLoggedIn && <h1>Welcome back!</h1>}
</div>
```

## 5. 라우팅은 어떻게 처리 하는가?

- react-router-dom 라이브러리를 사용하면 됩니다.
- BrowserRouter로 전체 컴포넌트를 감싸 줍니다.
- BrowserRouter 안에 Routes 컴포넌트를 넣어줍니다.
- Routes 컴포넌트 안에 이동하고자 하는 경로를 Route 컴포넌트로 만들어 넣어 줍니다.

<br />

- 렌더링할 컴포넌트들이 표시되는 영역입니다. Outlet의 위치에 해당 페이지의 컴포넌트들이 렌더링 됩니다.

## 6. redux는 무엇이고 어떻게 동작하는가?

- 리덕스는 전역 상태 관리 라이브러리입니다.
- 기존의 단방향식 데이터 흐름을 양방향으로 바꾸어 줍니다.
- redux는 store, dispatch, action, reducer 4가지 개념으로 나누어 설명할 수 있습니다.
- store : 애플리케이션의 상태를 저장하는 공간입니다.
- action : 무슨 작업이 이루어 지는지 설명하는 객체입니다. 인자를 넣을 수 있습니다.
- dispatch : reducer로 action 객체를 전달하는 함수 입니다.
- reducer: dispatch로부터 전달받은 액션 객체에 따라 상태를 변화시킵니다.

<br />

### 6-1. redux saga는 무엇인가?

- 리덕스 사가는 리덕스에서 비동기 작업을 처리하기 위한 미들웨어 입니다. 서버에서 데이터를 받아 store에 상태 값으로 저장하는 경우를 예로 들 수 있습니다.
- 리덕스 사가는 제네레이터 문법을 사용해 함수의 진행 흐름을 제어할 수 있습니다. 예를들어, yield 키워드를 사용해, 서버에서 데이터를 가져올 때까지 함수의 진행을 멈춥니다. <br />
- 데이터를 받아온 이후에 yield 키워드를 통해 상태 값을 변경하는 과정을 뒤이어 진행할 수 있습니다.

<br />

### 6-2. 미들웨어란 무엇인가?

- 액션과 리듀서 사이에서 동작하는 일종의 중간 단계로, 특정한 작업을 수행하게끔 할 수 있습니다.
- 상태가 변경되기 이전에 다른 작업이 필요하다면 미들웨어를 이용하면 됩니다.

<br />

### 6-3. 제네레이터 문법은 무엇인가?

- es6에서 새로 추가된 문법입니다.
- function* 키워드를 사용해 생성합니다.
- yield 키워드, next() 메서드를 사용해, 제어 흐름을 일시 중지 할 수 있습니다. <br />
yield 키워드를 사용하면 해당 코드 이후의 코드를 실행되지 않습니다. <br />
next()메서드를 사용하면 이후의 코드가 다시 실행됩니다. <br />

<br />

### 6-4. 리덕스 툴킷은 무엇인가?

- 리덕스의 코드를 좀 더 줄여 생산성을 높이는 라이브러리 입니다. 줄여서 RTK라고도 부릅니다.
- 리덕스 툴킷의 주요 기능은 다음과 같습니다.
- configureStore : 스토어 생성을 간소화하고, 기본적인 미들웨어를 자동으로 추가해줍니다. redux devtools를 따로 설치할 필요가 없습니다.
- createReducer: Immer 라이브러리를 활용하여 불변성을 유지시킵니다. 툴킷을 사용하지 않을 경우, immer 라이브러리를 사용할 수 있습니다.
- createAction: 액션 생성자 함수를 좀더 간편하게 생성 가능해집니다.
- createSlice: redux의 상태와 액션 생성자 함수를 묶어서 작성할 수 있습니다.
